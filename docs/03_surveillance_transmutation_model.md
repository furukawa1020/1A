# Surveillance Transmutation Model / 監視化転化モデル

## English

The model describes how a system introduced for self-support can become surveillance or imposed labeling.

### Model Components

| Component | Role |
| --- | --- |
| Support intention | The stated purpose, such as early noticing or self-observation. |
| Sensing / logging | Collection of state-related work or study data. |
| State inference | Computation that interprets behavioral traces as internal state. |
| Output claim | The strength of what the system says about the user. |
| Visibility / sharing | Who can access the resulting output. |
| Effect | Monitoring feeling, label imposition, disclosure chilling, or self-observation support. |

### Design Claim

The model treats non-assertive output not as a nicer wording style, but as a restriction on system claims. The system should avoid presenting itself as an authority over the user's internal state when the research goal is self-observation support.

## 日本語

このモデルは、本人支援のために導入されたシステムが、どのように監視またはラベル押し付けへ転化しうるかを記述する。

### モデル構成要素

| 構成要素 | 役割 |
| --- | --- |
| 支援意図 | 早期気づきや自己観察など、表向きの目的。 |
| センシング / ログ化 | 仕事・学業に関する状態関連データの収集。 |
| 状態推定 | 行動痕跡を内部状態として解釈する計算。 |
| 出力主張 | システムがユーザーについて何をどの強さで述べるか。 |
| 可視性 / 共有 | 出力結果に誰がアクセスできるか。 |
| 効果 | 監視感、ラベル押し付け感、開示萎縮、自己観察支援。 |

### 設計上の主張

このモデルでは、非断定的出力を単なる優しい言い換えではなく、システムの主張を制限する設計として扱う。研究目的が自己観察支援であるなら、システムはユーザーの内部状態に対する権威として振る舞うことを避けるべきである。
