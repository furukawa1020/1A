# PRESENCE Framework / PRESENCEフレームワーク

## English

PRESENCE treats claims as security/privacy assets. The framework audits how signals become claims and how those claims cross authority and interpretation boundaries.

## Components

- Claim-flow graph: signal -> feature -> state estimate -> label -> claim -> recommendation -> administrative action.
- Authority boundary: self, supporter/researcher, teacher/manager, employer/evaluator, administrative action.
- Interpretation boundary: user interpretation, system suggestion, system label, administrative flag.
- Claim severity ladder: C0 through C6.
- Surveillance transmutation pattern catalog: P1 through P12.
- Machine-readable YAML audit spec.
- CLI report generator.

## 日本語
